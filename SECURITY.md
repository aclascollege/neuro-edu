## 🔒 Reporting a Vulnerability

The **Atlanta College of Liberal Arts and Sciences (ACLAS)** takes the security of our open-source software seriously.

If you discover a security vulnerability within ACLAS Neuro-Edu, please report it responsibly:

1. **Email**: Send a detailed report to **[info@aclas.college](mailto:info@aclas.college)** with the subject line `[SECURITY] Neuro-Edu Vulnerability Report`.
2. **Do NOT** open a public GitHub Issue for security vulnerabilities.
3. Include as much detail as possible: steps to reproduce, affected components, potential impact.

### What to Expect

- **Acknowledgment**: We will acknowledge receipt of your report within **48 hours**.
- **Assessment**: Our engineering team will assess the severity and impact within **5 business days**.
- **Resolution**: Critical vulnerabilities will be patched and released as soon as possible, with credit given to the reporter (if desired).

### Scope

This policy applies to the `aclascollege/neuro-edu` repository and any code deployed at [aclascollege.github.io/neuro-edu](https://aclascollege.github.io/neuro-edu/).
