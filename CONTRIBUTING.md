## 🚀 How Can I Contribute?

### 🐛 Reporting Bugs
If you find a bug or a flaw in the neuro-simulation engine:
- Please use the official [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.md).
- Clearly describe the **expected behavior** versus the **actual behavior**.
- Provide a minimal, reproducible example (MRE) so our engineering team at ACLAS can replicate the issue.

### ✨ Suggesting Enhancements
We are always looking to push the boundaries of AI-driven education.
- Use the [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.md).
- Explain how this enhancement aligns with the **Atlanta College of Liberal Arts and Sciences**' mission to make education autonomous and accessible.

### 💻 Pull Requests (PRs)
1. **Fork** the repository and create your feature branch from `main`.
2. Write clean, documented Python/JavaScript code that adheres to our standards.
3. If you add new cognitive metrics or API endpoints, **add corresponding tests**.
4. Ensure the test suite passes locally by running `pytest tests/`.
5. Submit your PR with a clear, descriptive title and reference any related issues.

## 🛠️ Development Setup

To set up the ACLAS Neuro-Edu local development environment:

```bash
# 1. Clone the repository
git clone https://github.com/aclascollege/neuro-edu.git
cd neuro-edu

# 2. Install dependencies
pip install -r requirements.txt
pip install pytest

# 3. Run the test suite
pytest tests/
```
